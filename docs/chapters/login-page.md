# Login page

Most NORA instances start at a login page before opening project data. The page provides the authenticated entry point for users and a few unauthenticated helper actions.

![NORA login page](../assets/images/gallery/login/nora-login-panel.jpg)

## Fields and actions

- **Username / Password:** sign in with a NORA account or an institution-backed account, depending on the local instance configuration.
- **Login:** submits the credentials and opens the NORA application.
- **Forgot password?:** starts the password reset flow for accounts managed by the instance.
- **Standalone Viewer:** opens the viewer without entering a project. This is useful for loading local files or testing viewer behavior.
- **Create new account:** opens the account request or registration flow, if enabled for the instance.

After login, NORA can resume automatically if another browser tab already has an active session.
